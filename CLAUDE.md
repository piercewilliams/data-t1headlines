## Analysis Governor — mandatory at every session

**Read `GOVERNOR.md` at the start of every session before doing any analysis or answering any analysis question.** The governor has two parts that are non-negotiable:

### Part 1 — Relevance (what to pursue)
- Check `## Stakeholder Focus` — shape what you surface toward current priorities
- Check `## Active Probing Queue` — HIGH priority items should be run on ingest events even if not asked
- Check `## Confirmed Low-Signal Areas` — do not probe these without an explicit user ask
- Check `## What "Interesting" Means` — filter findings through this calibration standard before presenting

### Part 2 — Rigor (how to pursue it)
**`GOVERNOR.md` Part 2 is the authoritative source for all rigor rules.** The summary below is a quick reference; if it ever conflicts with GOVERNOR.md, GOVERNOR.md wins.

Every finding you present — whether in response to a question or proactively surfaced — **must satisfy all Required Fields (full spec in GOVERNOR.md § Required Fields):**
```
Source:   sheet + column + filters applied + n after filters
Test:     which test + why appropriate for this data type
Result:   test statistic + exact p-value + effect size r + CI if available
Baseline: comparison group + its n + its median
Tier:     significant (p<0.05) / directional (p<0.10) / no detectable difference (p≥0.10)
Scope:    what platform, year range, outlet scope this does and does not cover
```
A finding that cannot fill in all fields is **not ready to present**. Report it as "incomplete / cannot fully test" — never fill in missing values from memory, inference, or prior sessions.

**Language rules — enforced, not suggested (full table in GOVERNOR.md § Language Precision Standards):**
- "Significantly" = p<0.05 only. "Directionally" = p<0.10 only. Never "significantly" for p≥0.05.
- "Causes / drives / leads to" = never in observational data. Use "associated with."
- "Confirms" = pre-registered hypothesis only. Post-hoc exploratory findings use "suggests."
- Featured pick analyses: always explicitly separate editorial-selected views from organic algorithmic views.
- Multiple comparisons: when running ≥5 tests in a family, report raw p AND Bonferroni threshold α/k.

**Anti-hallucination — hard rules (full list in GOVERNOR.md § Anti-Hallucination Rules):**
- Never cite a specific number (n, p-value, median, lift) without having computed it this session
- SmartNews 2026 is domain-aggregated — formula/length analysis on it is invalid; say so explicitly
- Always verify `classify_formula()` baseline key = `"untagged"` before any formula comparison
- MSN analysis: use raw `"Pageviews"` column, not `percentile_within_cohort`
- Read GOVERNOR.md § Known Data Quirks before any analysis — silent failure modes are documented there

## New data ingest — run this autonomously

When the user provides new data (a file path, a file in Downloads, a new sheet, any combination), **execute the full ingest pipeline without asking for confirmation**. The steps:

### 1. Identify the file(s)
- If the user gives a path, use it directly.
- If they say "Downloads" or a filename without a path, check `~/Downloads/` for the most recently modified `.xlsx` file matching the description.
- The 2026 file is the one that changes monthly. The 2025 file is the historical baseline and only needs updating if the user says so.

### 2. Run ingest
```bash
python3 ingest.py --data-2026 "<path>" [--data-2025 "<path>"] [--release YYYY-MM]
```
- Add `--release YYYY-MM` if the user says the data is from a specific past month.
- Default (no `--release`) uses current month — correct for same-month runs.

### 3. Read the build report and fix problems autonomously

The build report prints at the end. Handle each output type:

**Column validation failure** (`✗ Missing column '...'`):
- The error shows available columns. Find the closest match, update the column name in `generate_site.py`, re-run. Don't ask — fix and re-run.

**Sheet discovery warning** (new sheet not in pipeline):
- Read the sheet briefly with pandas to assess row count and columns.
- If it looks like data worth analyzing (>50 rows, recognizable metrics), note it for the user but don't block — add it to `_KNOWN_SHEETS_2026` in `generate_site.py` so it stops warning, and flag it for future pipeline work.
- If it looks like a summary/pivot (few rows), just add to known list and move on.

**Rigor warnings** (subtopic tables without significance tests):
- These are expected. The 3 standing ones (sports/biz/pol subtopics) are known and tracked. Don't treat as errors.

**Row count sanity**: if Apple News or SmartNews rows are *lower* than a prior run (check `docs/archive/*/meta.json` for previous counts), warn the user before committing — the export may be truncated.

### 4. Confirm done
Tell the user:
- What was archived (which month's analysis)
- Row counts for this run vs. last run (from `meta.json` comparison)
- Playbook tile confidence breakdown
- Any new sheets found and what you decided about them
- "Ready to push from GitHub Desktop"

## Autonomy boundaries

**Do without asking:**
- Run `ingest.py` and `generate_site.py`
- Fix column renames when the new name is unambiguous from the available columns list
- Add newly discovered sheets to the known-sheets list (so they stop warning)
- Re-run after fixing errors

**Ask before doing:**
- Wiring a new sheet into the full analysis pipeline (requires understanding the schema and writing new analysis code — get user's go-ahead first)
- Committing if the build fails (archive is preserved; don't commit a broken state)

## Code structure notes (relevant for edits)

**Nav is programmatic — do not hardcode it.** All four pages (main, playbook, author-playbooks, experiments) use `_build_nav()` from their respective generator script. `_NAV_PAGES` in `generate_site.py` is the single source of truth for site structure. To add a new nav page: update `_NAV_PAGES`, re-run both generators.

**Export JS is programmatic — do not hardcode it.** Each page's PNG/PDF export block is generated by `_make_export_js(suffix, tile_cleanup_selector, heading_query, filename_prefix, find_tile_body)`. To change export behavior across all pages: edit `_make_export_js`. To change only one page's behavior: update that page's call arguments.

## Python environment

All dependencies are in `requirements.txt`. On a new machine:
```bash
pip3 install -r requirements.txt
```

The build report shows which optional packages are active (statsmodels ✓/✗, sklearn ✓/✗, polars ✓/✗, pingouin ✓/✗). If any show ✗, the pipeline still runs but with reduced analytical capability — install missing packages to restore full output.
