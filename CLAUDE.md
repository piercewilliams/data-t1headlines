# T1 Headline Analysis — Claude Code Instructions

## What this project is
Monthly analysis of McClatchy T1 outlet headline performance across Apple News, SmartNews, MSN, and Yahoo. Data comes from Chris Tarrow's Google Sheet exports (Excel). The site lives at `docs/` and is generated by `generate_site.py`. `ingest.py` is the canonical entry point for new data.

For full project context: `REFERENCE.md`. For current status: `CONTEXT.md`.

## New data ingest — run this autonomously

When the user provides new data (a file path, a file in Downloads, a new sheet, any combination), **execute the full ingest pipeline without asking for confirmation**. The steps:

### 1. Identify the file(s)
- If the user gives a path, use it directly.
- If they say "Downloads" or a filename without a path, check `~/Downloads/` for the most recently modified `.xlsx` file matching the description.
- The 2026 file is the one that changes monthly. The 2025 file is the historical baseline and only needs updating if the user says so.

### 2. Run ingest
```bash
python3 ingest.py --data-2026 "<path>" [--data-2025 "<path>"] [--release YYYY-MM]
```
- Add `--release YYYY-MM` if the user says the data is from a specific past month.
- Default (no `--release`) uses current month — correct for same-month runs.

### 3. Read the build report and fix problems autonomously

The build report prints at the end. Handle each output type:

**Column validation failure** (`✗ Missing column '...'`):
- The error shows available columns. Find the closest match, update the column name in `generate_site.py`, re-run. Don't ask — fix and re-run.

**Sheet discovery warning** (new sheet not in pipeline):
- Read the sheet briefly with pandas to assess row count and columns.
- If it looks like data worth analyzing (>50 rows, recognizable metrics), note it for the user but don't block — add it to `_KNOWN_SHEETS_2026` in `generate_site.py` so it stops warning, and flag it for future pipeline work.
- If it looks like a summary/pivot (few rows), just add to known list and move on.

**Rigor warnings** (subtopic tables without significance tests):
- These are expected. The 3 standing ones (sports/biz/pol subtopics) are known and tracked. Don't treat as errors.

**Row count sanity**: if Apple News or SmartNews rows are *lower* than a prior run (check `docs/archive/*/meta.json` for previous counts), warn the user before committing — the export may be truncated.

### 4. Confirm done
Tell the user:
- What was archived (which month's analysis)
- Row counts for this run vs. last run (from `meta.json` comparison)
- Playbook tile confidence breakdown
- Any new sheets found and what you decided about them
- "Ready to push from GitHub Desktop"

## Autonomy boundaries

**Do without asking:**
- Run `ingest.py` and `generate_site.py`
- Fix column renames when the new name is unambiguous from the available columns list
- Add newly discovered sheets to the known-sheets list (so they stop warning)
- Re-run after fixing errors

**Ask before doing:**
- Wiring a new sheet into the full analysis pipeline (requires understanding the schema and writing new analysis code — get user's go-ahead first)
- Committing if the build fails (archive is preserved; don't commit a broken state)
- Anything that would overwrite or delete archived files

## Python environment

All dependencies are in `requirements.txt`. On a new machine:
```bash
pip3 install -r requirements.txt
```

The build report shows which optional packages are active (statsmodels ✓/✗, sklearn ✓/✗, polars ✓/✗, pingouin ✓/✗). If any show ✗, the pipeline still runs but with reduced analytical capability — install missing packages to restore full output.
